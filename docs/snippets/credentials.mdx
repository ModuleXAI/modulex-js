---
title: "Credentials"
description: "Store and manage authentication credentials for integrations, LLM providers, and MCP servers. Supports API keys, bearer tokens, ModuleX-managed keys, and custom auth."
---

## Installation

```bash
npm install modulex
```

## Initialize Client

```typescript
import { Modulex } from 'modulex-js';

const client = new Modulex({
  apiKey: 'mx_live_...',
  organizationId: 'your-org-id',
});
```

## list

Returns stored credentials for the organization. Without a filter the response is grouped by integration; with `integrationName` it returns a flat list.

<ParamField path="params.integrationName" type="string" optional>
  Filter to a single integration (e.g., `"openai"`, `"github"`). When provided the response is a flat credential list instead of the grouped object.
</ParamField>

<ParamField path="params.authType" type="string" optional>
  Filter by auth type: `"api_key"`, `"bearer_token"`, `"oauth2"`, `"modulex_key"`, `"custom"`.
</ParamField>

<ParamField path="params.limit" type="number" optional>
  Max results. Range: 1–500. Default: 100.
</ParamField>

<ParamField path="params.offset" type="number" optional>
  Pagination offset. Default: 0.
</ParamField>

```typescript
// Grouped by integration
const grouped = await client.credentials.list();
console.log(grouped.total_credentials);   // 10
console.log(grouped.total_integrations);  // 5
for (const [name, group] of Object.entries(grouped.integrations)) {
  console.log(`${name}: ${group.total_count} credential(s)`);
}

// Flat list for one integration
const openaiCreds = await client.credentials.list({ integrationName: 'openai' });
console.log(openaiCreds.total_count);    // 2
for (const cred of openaiCreds.credentials) {
  console.log(`${cred.display_name} — default: ${cred.is_default}`);
}
```

## get

Returns full credential metadata. Sensitive auth data is always masked unless `includeMasked` is explicitly requested.

<ParamField path="credentialId" type="string" required>
  UUID of the credential.
</ParamField>

<ParamField path="params.includeMasked" type="boolean" optional>
  Include masked (partially redacted) sensitive fields. Default: `false`.
</ParamField>

```typescript
const cred = await client.credentials.get('cred-uuid-here', { includeMasked: true });

console.log(cred.credential_id);      // "cred-uuid"
console.log(cred.integration_name);   // "openai"
console.log(cred.integration_type);   // "llm_provider"
console.log(cred.display_name);       // "My OpenAI Key"
console.log(cred.auth_type);          // "api_key"
console.log(cred.is_default);         // true
console.log(cred.last_used_at);       // "2026-03-09T12:00:00Z"
```

## create

Stores a new credential for an integration. Auth type is inferred from `authData` shape unless explicitly set.

### API Key

```typescript
const cred = await client.credentials.create({
  integrationName: 'openai',
  authData: { api_key: 'sk-proj-xxxxxxxxxxxxxxxx' },
  displayName: 'OpenAI Production Key',
  makeDefault: true,
});

console.log(cred.credential_id);  // "cred-uuid"
console.log(cred.auth_type);      // "api_key"
console.log(cred.is_default);     // true
```

### Bearer Token

```typescript
const cred = await client.credentials.create({
  integrationName: 'github',
  authData: { token: 'ghp_xxxxxxxxxxxxxxxxxxxx' },
  displayName: 'GitHub Actions Token',
  expiresAt: '2027-01-01T00:00:00Z',
  makeDefault: false,
});
```

### ModuleX Key (system-managed)

ModuleX manages the credential lifecycle — no `authData` required.

```typescript
const cred = await client.credentials.create({
  integrationName: 'openai',
  authType: 'modulex_key',
  displayName: 'ModuleX Managed OpenAI',
  makeDefault: false,
});

console.log(cred.auth_type);  // "modulex_key"
```

### OAuth2

```typescript
const cred = await client.credentials.create({
  integrationName: 'google',
  authData: { access_token: 'ya29.xxxx', refresh_token: 'xxxxx' },
  oauthConfig: { client_id: 'xxx.apps.googleusercontent.com', scope: 'email profile' },
  displayName: 'Google Workspace',
  makeDefault: true,
});
```

### Custom Auth

```typescript
const cred = await client.credentials.create({
  integrationName: 'my_internal_service',
  authType: 'custom',
  authData: {
    endpoint: 'https://internal.example.com/api',
    secret_key: 'svc_xxxxxxxxxxxxx',
  },
  displayName: 'Internal Service Auth',
});
```

### MCP Server

```typescript
const cred = await client.credentials.create({
  integrationName: 'mcp_server',
  authData: {
    server_url: 'https://mcp.example.com/sse',
    headers: { Authorization: 'Bearer svc_token_xxx' },
  },
  displayName: 'Internal MCP Server',
});
```

## update

Updates a credential's display name or metadata. Auth data cannot be changed — create a new credential instead.

<ParamField path="credentialId" type="string" required>
  UUID of the credential to update.
</ParamField>

<ParamField path="params.displayName" type="string" optional>
  New human-readable name.
</ParamField>

<ParamField path="params.metadata" type="object" optional>
  Arbitrary key-value metadata to attach to the credential.
</ParamField>

```typescript
await client.credentials.update('cred-uuid-here', {
  displayName: 'OpenAI Production Key (rotated 2026-03)',
  metadata: { rotated_by: 'alice', rotation_reason: 'quarterly rotation' },
});
```

## delete

Permanently deletes a credential. Returns HTTP 204 with no response body on success.

<ParamField path="credentialId" type="string" required>
  UUID of the credential to delete.
</ParamField>

```typescript
await client.credentials.delete('cred-uuid-here');
console.log('Credential deleted.');
```

## setDefault

Sets this credential as the default for its integration. Any existing default for the same integration is automatically unset.

<ParamField path="credentialId" type="string" required>
  UUID of the credential to promote to default.
</ParamField>

```typescript
const cred = await client.credentials.setDefault('cred-uuid-here');
console.log(cred.is_default);         // true
console.log(cred.integration_name);   // "openai"
```

## testTemporary

Validates a credential without persisting it. Useful for checking user-provided keys before saving.

<ParamField path="params.integrationName" type="string" required>
  Integration to test against.
</ParamField>

<ParamField path="params.authType" type="string" required>
  Auth type: `"api_key"`, `"bearer_token"`, `"oauth2"`.
</ParamField>

<ParamField path="params.authData" type="object" required>
  Credential data to test.
</ParamField>

```typescript
const result = await client.credentials.testTemporary({
  integrationName: 'openai',
  authType: 'api_key',
  authData: { api_key: 'sk-proj-candidate-key' },
});

if (result.is_valid) {
  console.log(`Valid! Tested via ${result.test_method} in ${result.status_code}`);
} else {
  console.error(`Invalid: ${result.message}`);
}
console.log(`Cost level: ${result.cost_level}`);  // "low" | "free" | null
```

## test

Tests an already-stored credential by running a live validation request against the integration.

<ParamField path="credentialId" type="string" required>
  UUID of the stored credential to test.
</ParamField>

```typescript
const result = await client.credentials.test('cred-uuid-here');

console.log(result.credential_id);  // "cred-uuid"
console.log(result.is_valid);       // true
console.log(result.message);        // "Connection successful"
console.log(result.tested_at);      // "2026-03-09T14:30:00Z"
```

## usage

Returns aggregated usage statistics for a credential over a given time period.

<ParamField path="credentialId" type="string" required>
  UUID of the credential.
</ParamField>

<ParamField path="params.startDate" type="string" optional>
  ISO 8601 start of the reporting period.
</ParamField>

<ParamField path="params.endDate" type="string" optional>
  ISO 8601 end of the reporting period.
</ParamField>

```typescript
const usage = await client.credentials.usage('cred-uuid-here', {
  startDate: '2026-03-01T00:00:00Z',
  endDate: '2026-03-09T23:59:59Z',
});

console.log(usage.total_calls);       // 1000
console.log(usage.successful_calls);  // 990
console.log(usage.failed_calls);      // 10
console.log(usage.success_rate);      // 0.99
console.log(usage.action_breakdown);  // { "web_search": 500, "create_issue": 490 }
```

## audit

Returns a paginated audit log of all operations performed using this credential.

<ParamField path="credentialId" type="string" required>
  UUID of the credential.
</ParamField>

<ParamField path="params.limit" type="number" optional>
  Max log entries. Range: 1–500. Default: 100.
</ParamField>

<ParamField path="params.offset" type="number" optional>
  Pagination offset. Default: 0.
</ParamField>

```typescript
const logs = await client.credentials.audit('cred-uuid-here', { limit: 50 });

for (const entry of logs) {
  console.log(`${entry.timestamp} — ${entry.operation} by ${entry.performed_by}`);
}
```

## bulkModulexKeys

Streams SSE progress events while ModuleX bulk-creates managed keys for all supported integrations. Use `for await...of` to track progress.

```typescript
for await (const event of client.credentials.bulkModulexKeys()) {
  switch (event.type) {
    case 'progress':
      console.log(`Creating key for: ${event.data.integration_name}`);
      break;
    case 'created':
      console.log(`Created: ${event.data.integration_name} — id: ${event.data.credential_id}`);
      break;
    case 'error':
      console.warn(`Failed: ${event.data.integration_name} — ${event.data.error}`);
      break;
    case 'done':
      console.log(`Bulk creation complete. Created: ${event.data.created_count}`);
      break;
  }
}
```

## mcpServer

Registers a new MCP server and discovers its available tools automatically.

<ParamField path="params.serverUrl" type="string" required>
  HTTPS URL of the MCP server endpoint.
</ParamField>

<ParamField path="params.headers" type="object" optional>
  Custom headers to include in every request to the MCP server (e.g., `Authorization`).
</ParamField>

<ParamField path="params.displayName" type="string" optional>
  Human-readable label for this server.
</ParamField>

```typescript
const mcpCred = await client.credentials.mcpServer({
  serverUrl: 'https://mcp.internal.example.com/sse',
  headers: { Authorization: 'Bearer internal-token-xxx' },
  displayName: 'Internal Data MCP Server',
  makeDefault: false,
});

console.log(mcpCred.credential_id);  // "cred-uuid"
console.log(mcpCred.total_tools);    // 7 — tools discovered at registration
```

## refreshDiscovery

Re-connects to an MCP server and refreshes its tool catalog. Use this after updating the server's tool definitions.

<ParamField path="credentialId" type="string" required>
  UUID of the MCP server credential to refresh.
</ParamField>

```typescript
const result = await client.credentials.refreshDiscovery('cred-uuid-here');

console.log(result.success);       // true
console.log(result.total_tools);   // 9 — updated tool count
console.log(result.refreshed_at);  // "2026-03-09T15:00:00Z"
console.log(result.changes);       // Diff of added/removed tools
```

## mcpTools

Returns the list of tools currently registered for an MCP server credential.

<ParamField path="credentialId" type="string" required>
  UUID of the MCP server credential.
</ParamField>

```typescript
const result = await client.credentials.mcpTools('cred-uuid-here');

console.log(result.credential_id);  // "cred-uuid"
console.log(result.total_count);    // 9

for (const tool of result.tools) {
  console.log(`${tool.name}: ${tool.description}`);
}
```
